6class Choices(object): 

7 """ 

8 A class to encapsulate handy functionality for lists of choices 

9 for a Django model field. 

10 

11 Each argument to ``Choices`` is a choice, represented as either a 

12 string, a two-tuple, or a three-tuple. 

13 

14 If a single string is provided, that string is used as the 

15 database representation of the choice as well as the 

16 human-readable presentation. 

17 

18 If a two-tuple is provided, the first item is used as the database 

19 representation and the second the human-readable presentation. 

20 

21 If a triple is provided, the first item is the database 

22 representation, the second a valid Python identifier that can be 

23 used as a readable label in code, and the third the human-readable 

24 presentation. This is most useful when the database representation 

25 must sacrifice readability for some reason: to achieve a specific 

26 ordering, to use an integer rather than a character field, etc. 

27 

28 Regardless of what representation of each choice is originally 

29 given, when iterated over or indexed into, a ``Choices`` object 

30 behaves as the standard Django choices list of two-tuples. 

31 

32 If the triple form is used, the Python identifier names can be 

33 accessed as attributes on the ``Choices`` object, returning the 

34 database representation. (If the single or two-tuple forms are 

35 used and the database representation happens to be a valid Python 

36 identifier, the database representation itself is available as an 

37 attribute on the ``Choices`` object, returning itself.) 

38 

39 Option groups can also be used with ``Choices``; in that case each 

40 argument is a tuple consisting of the option group name and a list 

41 of options, where each option in the list is either a string, a 

42 two-tuple, or a triple as outlined above. 

43 

44 """
